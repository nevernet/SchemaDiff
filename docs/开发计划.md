### 1.2 修复外键 (FK) 解析
**具体功能点**:
- 正确解析 FOREIGN KEY 约束的引用表和列
- 支持 ON DELETE / ON UPDATE 动作
- 支持表级和列级外键语法

**当前问题**:
```python
# 当前解析结果
FK: fk_auto -> ([])  # ref_table 和 ref_columns 都为空
```

**实现思路**:
1. 检查 `_parse_foreign_key` 函数中 `reference` 属性的访问方式
2. 使用 `kind.sql()` 获取完整的 FK 定义
3. 调试 sqlglot 返回的 AST 结构

**修复代码思路**:
```python
def _parse_foreign_key(fk_expr: exp.ForeignKey, dialect: str) -> Optional[ForeignKey]:
    ref = fk_expr.args.get("reference")
    if not ref:
        return None
    
    # 正确获取引用表名
    ref_table = ref.name if hasattr(ref, "name") else ref.sql(dialect=dialect)
    
    # 正确获取引用列
    ref_columns = []
    if hasattr(ref, "columns"):
        for c in ref.columns:
            ref_columns.append(c.name if hasattr(c, "name") else c.sql(dialect=dialect))
    
    # 解析 ON DELETE / ON UPDATE
    on_delete = fk_expr.args.get("ondelete")
    on_update = fk_expr.args.get("onupdate")
```

**优先级**: P0
